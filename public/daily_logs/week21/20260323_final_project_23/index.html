2. 실습 내용 정리#

활동 1: 주요 산출물 최종 마무리#

목표 : BIG 산출물(WBS, 수행계획서, 위수탁 계약서, 수탁사 목록, 사업제안서) 오늘 내 완료

우선순위 기준 :

-  WBS 일정·담당자·항목명 최종 정합성 확인
-  수행계획서 날짜·담당자 수정 완료 여부
-  위수탁 계약서 보안 요구사항 포함 여부
-  수탁사 목록 항목 누락 없는지 확인
-  사업제안서 수치·날짜 사실 확인 완료 여부

보안 인사이트 :

  • BIG 산출물이 먼저 완료되어야 발표 PPT의 현재 진행도 슬라이드가 정확하게 채워짐
  • 산출물 간 날짜·항목명 정합성은 발표 Q&A에서 바로 드러나는 부분

6. 배운 점 및 인사이트#

새로 알게 된 점#

  • 주간 목표를 요일별로 쪼개는 것의 힘 : 막연하게 “이번 주 안에 끝내자"가 아니라 월·화·수·목·금 각각에 명확한 목표를 부여하면 하루가 끝날 때 달성 여부를 즉시 점검할 수 있음

  • 병렬 작업의 효율 : PPT 작업과 산출물 마무리를 동시에 돌리는 것이 가능한 이유는 담당자가 명확히 분리되어 있기 때문. Day 95에서 확정된 담당자 배정이 지금 빛을 발하는 순간

이전 학습과의 연결고리#

  • Day 95 담당자 배정과 연계 : 역할 분리가 명확했기에 오늘부터 병렬 작업이 가능

  • Day 96 프로세스 페르소나와 연계 : 발표 구성 전체가 “우리는 올바른 프로세스를 밟았다"는 메시지 중심으로 설계됨

  • Day 94 일정 기준과 연계 : 확정된 프로젝트 일정이 발표 PPT의 진행도·잔여 일정 슬라이드 기준으로 그대로 사용됨

실무 적용 아이디어#

보안 전문가 관점 :

-  마감 역산 관리 : 발표일(목요일)을 기준으로 역산해서 수·화·월에 무엇이 완료되어야 하는지를 먼저 확정하는 방식은 실무 프로젝트 관리에서도 동일하게 작동함

-  전수 피어 리뷰의 타이밍 : 수요일처럼 발표 전날 전수 교차 검증을 넣는 것은 최종 제출 직전 품질 게이트를 두는 실무 관행과 동일
